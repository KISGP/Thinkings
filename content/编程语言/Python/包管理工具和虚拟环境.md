## pip

### 🧪 什么是 pip？

`pip` 是 Python 官方推荐的**包管理工具**，用于从 [PyPI](https://pypi.org/)（Python Package Index）安装和管理 Python 软件包。

从 Python 3.4+ 开始，`pip` 默认包含在安装包中。

### 🛠️ pip 的主要功能

| 功能         | 命令示例                              | 说明                             |
| ------------ | ------------------------------------- | -------------------------------- |
| 安装包       | `pip install requests`                | 从 PyPI 安装指定包               |
| 指定版本     | `pip install requests==2.31.0`        | 安装特定版本                     |
| 升级包       | `pip install --upgrade requests`      | 升级已安装包                     |
| 卸载包       | `pip uninstall requests`              | 删除指定包                       |
| 查看已安装   | `pip list`                            | 显示当前环境下已安装包           |
| 查看包信息   | `pip show requests`                   | 包的详细信息（路径、依赖等）     |
| 搜索包       | `pip search flask`（已废弃）          | 旧版搜索，现推荐直接上 PyPI 网站 |
| 冻结依赖     | `pip freeze > requirements.txt`       | 生成当前环境的依赖清单           |
| 安装依赖清单 | `pip install -r requirements.txt`     | 批量安装依赖包                   |
| 升级 pip     | `python -m pip install --upgrade pip` | 升级自身版本                     |

### 📁 pip 安装路径与缓存

- 安装路径：依赖你的 Python 环境（如系统 Python、虚拟环境）
- 缓存位置：
  - Linux/macOS: `~/.cache/pip`
  - Windows: `%LocalAppData%\pip\Cache`

### 🔄 pip 使用示例

```bash
# 安装最新版 flask
pip install flask

# 安装多个包
pip install numpy pandas matplotlib

# 安装并指定版本
pip install "Django>=3.2,<4.0"

# 从 Git 仓库安装
pip install git+https://github.com/pallets/flask.git

# 使用国内镜像源安装（如清华）
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple flask
```

# 虚拟环境

虚拟环境是一种**隔离的 Python 运行环境**，可让你为不同项目分别管理依赖包，避免包版本冲突，提升开发的整洁性和可维护性。

## venv

> 官方文档：[https://docs.python.org/zh-cn/3/library/venv.html](https://docs.python.org/zh-cn/3/library/venv.html)

### 🧪 什么是 venv？

`venv` 是 Python 3.3 及以上版本自带的**标准库模块**，用于创建**虚拟环境**。

### 🛠️ 使用 venv？

#### ✅ 创建虚拟环境

```bash
python -m venv myenv
```

> `myenv` 是你虚拟环境的目录名，也可以叫 `venv`、`.venv`，一般会放在项目根目录。

> 如果你不使用 `-m` 参数，而是像直接运行：`python venv myenv`，你会看到类似以下的错误：
>
> ```yaml
> python: can't open file 'venv': [Errno 2] No such file or directory
> ```
>
> `-m` 参数表示这是运行已安装模块（如标准库或 site-packages 中的模块）。不携带该参数表示这是尝试运行一个 `.py` 文件（必须在文件系统中存在）。
>
> 在 **Windows 系统下**，Python 的 `venv` 模块是**作为标准库的一部分**，存放在 Python 安装目录下的 `Lib` 文件夹中。
>
> ![image-20250507173640563](assets/image-20250507173640563.png)

#### ✅ 激活虚拟环境

| 平台    | Shell      | 用于激活虚拟环境的命令                |
| :------ | :--------- | :------------------------------------ |
| POSIX   | bash/zsh   | `$ source <venv>/bin/activate`        |
|         | fish       | `$ source <venv>/bin/activate.fish`   |
|         | csh/tcsh   | `$ source <venv>/bin/activate.csh`    |
|         | pwsh       | `$ <venv>/bin/Activate.ps1`           |
| Windows | cmd.exe    | `C:\> <venv>\Scripts\activate.bat`    |
|         | PowerShell | `PS C:\> <venv>\Scripts\Activate.ps1` |

下图是 Vscode 启动虚拟环境的结果。

![Vscode 启动虚拟环境](assets/image-20250507171332055.png)

#### ✅ 安装包（在虚拟环境中）

```bash
pip install requests
```

该操作只会安装到 `myenv` 环境下，不影响全局 Python。

#### ✅ 冻结依赖（用于部署或分享）

```bash
pip freeze > requirements.txt
```

#### ✅ 退出虚拟环境

```bash
deactivate
```



### 📁 目录结构

```yaml
# Linux/macOS
myenv/
├── bin/
│   ├── activate          # 激活脚本
│   └── python            # 虚拟环境下的 python 可执行文件
├── lib/
│   └── python3.x/
│       └── site-packages/  # 安装的所有依赖包
└── pyvenv.cfg             # 配置文件
```

```yaml
# Windows
myenv/
├── Scripts/
│   ├── activate          # 激活脚本
│   └── python            # 虚拟环境下的 python 可执行文件
├── lib/
│   └── site-packages/  	# 安装的所有依赖包
└── pyvenv.cfg            # 配置文件
```

### ❗venv 的缺陷和局限

1. 不能创建跨版本 Python 虚拟环境

   `venv` 只能创建**当前 Python 解释器版本**的虚拟环境。如果有多个 Python 版本（如 Python 3.8 和 3.11），就不能用 `venv` 切换版本。

2. 不能为 Python 2 创建虚拟环境

   `venv` 是 Python 3.3+ 的内置模块，不支持 Python 2。如果你在维护老旧项目（如 Python 2.7），则可以使用 `virtualenv`。

3. 功能简化，缺少高级选项

   `venv` 不提供：快速复制环境（如模板化虚拟环境）、快速清理不必要文件、挂载系统 site-packages 选项（虽然可以通过参数实现）

4. 兼容性略逊

   某些操作系统/工具链（特别是老旧平台、CI/CD 工具）可能更倾向使用 `virtualenv`，因为其更成熟、历史更久。一些 Python 工具（如 `tox`、`pipenv`）默认使用或推荐 `virtualenv`。

5. 不支持“嵌套”或集中管理虚拟环境

   `venv` 每次只能创建一个环境，管理多个项目的环境时不太方便。而且不像 `conda` 或 `pyenv` 这样的工具，可以集中查看、删除所有环境。

## virtualenv

> https://virtualenv.pypa.io/

![image-20250507195418551](assets/image-20250507195418551.png)

### 🧪 什么是 virtualenv？

`virtualenv` 是一个比 `venv` 更早出现的第三方工具，用于创建独立的 Python 虚拟环境，功能强大、兼容性广；而 `venv` 是在 Python 3.3 中引入的官方简化版，继承了 `virtualenv` 的核心思想，提供基本的环境隔离能力，轻便易用，但功能不如前者丰富。

### 📦 安装 virtualenv

`virtualenv` 并不自带，需要手动安装（一次性）：

```bash
pip install virtualenv
```

### 🛠️ 使用 virtualenv

#### ✅ 创建虚拟环境

```bash
virtualenv myenv
```

这会创建一个名为 `myenv` 的目录，其中包含完整的 Python 虚拟环境。

![image-20250507175836244](assets/image-20250507175836244.png)

#### ✅ 创建虚拟环境（指定 Python 版本）

在创建虚拟环境可以通过 `-p` 指定 Python 版本

```bash
virtualenv -p C:\Python38\python.exe myenv
```

或者在 Unix 系统下：

```bash
virtualenv -p python3.8 myenv
```

> 这比 `venv` 更灵活，`venv` 只能使用当前解释器。

创建虚拟环境时**不加 `-p` 指定 Python 版本**，`virtualenv` 会使用**当前终端默认的 Python 解释器版本**来创建虚拟环境。

#### ✅ 激活环境

- **Windows：**

```bash
myenv\Scripts\activate
```

- **Linux/macOS：**

```bash
source myenv/bin/activate
```

> 激活后，终端会出现 `(myenv)` 前缀，表示你处于这个虚拟环境中。

#### ✅ 安装依赖

```bash
pip install flask
```

这些包都会安装在 `myenv` 目录下，不会影响全局 Python。

#### ✅ 退出虚拟环境

```bash
deactivate
```

### 📁 目录结构

```yaml
# Windows
myenv/
├── Scripts/
│   ├── activate.bat
│   ├── python.exe
│   └── pip.exe
├── Lib/
│   └── site-packages/  # 所有依赖包装在这里
├── pyvenv.cfg
```

### 📝 推荐使用场景

使用 `virtualenv` 的情况：

- 你需要支持 **多个 Python 版本**
- 你在维护 **Python 2 和 Python 3** 的兼容代码
- 你希望使用 `tox`, `pipenv`, `build`, `hatch` 等工具
- 你在 CI/CD 环境中需要灵活管理多个解释器

## pipenv

> https://pipenv.pypa.io/zh-cn/latest/

![image-20250507195531621](assets/image-20250507195531621.png)



### 🧪 什么是 pipenv？

**Pipenv** 是 Python 官方推荐的一种**集成“虚拟环境 + 依赖管理”的开发工具**，由 Python Packaging Authority（PyPA）开发，目标是替代 `requirements.txt` + `virtualenv` 的组合，使 Python 项目管理更一致、自动化、现代化。

### 📦 安装 pipenv

> 安装文档：https://pipenv.pypa.io/zh-cn/latest/install.html#installing-pipenv

安装 pipenv 有四种方式：通过 pipx、普通安装、通过 curl 直接安装、通过 Homebrew 安装。

1. pipx

   ```bash
   pip install --user pipx
   
   pipx install pipenv
   ```

2. 普通安装

   ```bash
   pip install --user pipenv
   ```

   安装后按照[文档教程](https://pipenv.pypa.io/zh-cn/latest/install.html#pragmatic-installation-of-pipenv)添加环境变量即可。

### 🛠️ 使用 pipenv

#### ✅初始化项目 + 创建虚拟环境

```bash
mkdir myproject
cd myproject
pipenv install
```

默认会使用当前系统默认的 Python 版本来创建 `.venv` 虚拟环境，并在项目中生成：`Pipfile`、`Pipfile.lock`

![image-20250507202525917](assets/image-20250507202525917.png)

#### ✅ 安装依赖

```bash
# 安装一个普通依赖
pipenv install requests

# 安装开发依赖（如测试、格式化工具）
pipenv install --dev black
```

自动生成或更新 `Pipfile` 和 `Pipfile.lock`。

#### ✅ 启动虚拟环境

```bash
pipenv shell
```

#### ✅ 查看依赖树

```bash
pipenv graph
```

#### ✅ 检查依赖安全性

```bash
pipenv check
```

✅ 指定 Python 版本（如果你有多个版本）

```bash
pipenv --python 3.10
```

#### ✅ 删除虚拟环境（重置环境）

```bash
pipenv --rm
```

## poetry

> https://python-poetry.org/

![image-20250507195833393](assets/image-20250507195833393.png)

#### 🧪 什么是 Poetry？

**Poetry** 是一个现代化的 **Python 项目管理和打包工具**，集成了：

- 虚拟环境管理
- 依赖管理（安装/卸载/锁定）
- 构建/发布 Python 包
- 与 `pyproject.toml` 配合，符合 PEP 517/518 标准

它旨在**替代 `pip` + `venv` + `setuptools` + `twine` 等多个工具组合**，统一管理你的 Python 项目。

Poetry 的核心目标是：

- ✅ 简化 Python 项目管理流程
- ✅ 提供一致性（所有元数据、依赖写在 `pyproject.toml`）
- ✅ 更快、更安全的依赖解析（比 pipenv 更稳定）
- ✅ 完善的打包和发布流程（可以一键发布到 PyPI）

#### 📦 安装 Poetry

推荐使用官方脚本安装（跨平台）：

```bash
curl -sSL https://install.python-poetry.org | python3 -
```

或者使用 pip（不推荐）：

```bash
pip install poetry
```

安装后确认版本：

```bash
poetry --version
```

#### 🛠️ Poetry 的主要功能

| 功能           | 命令                           | 说明                            |
| -------------- | ------------------------------ | ------------------------------- |
| 创建新项目     | `poetry new myproject`         | 包含项目结构和 `pyproject.toml` |
| 初始化现有项目 | `poetry init`                  | 生成 `pyproject.toml`           |
| 安装依赖       | `poetry add requests`          | 添加并更新 lock 文件            |
| 安装开发依赖   | `poetry add --group dev black` | 管理分组依赖（PEP 708）         |
| 移除依赖       | `poetry remove requests`       | 删除依赖                        |
| 启动虚拟环境   | `poetry shell`                 | 自动创建/激活虚拟环境           |
| 执行命令       | `poetry run python script.py`  | 在虚拟环境中运行命令            |
| 安装所有依赖   | `poetry install`               | 安装 lock 中的所有依赖          |
| 打包项目       | `poetry build`                 | 生成 `.tar.gz` 和 `.whl` 包     |
| 发布到 PyPI    | `poetry publish --build`       | 自动上传项目                    |
| 查看依赖树     | `poetry show --tree`           | 类似 `pipenv graph`             |

------

#### 📁 目录结构

```bash
myproject/
├── pyproject.toml        # 核心配置文件
├── myproject/
│   └── __init__.py
├── tests/
│   └── test_myproject.py
└── README.rst
```

#### 🧾 pyproject.toml 示例

```toml
[tool.poetry]
name = "myproject"
version = "0.1.0"
description = "An example project"
authors = ["Your Name <you@example.com>"]
license = "MIT"

[tool.poetry.dependencies]
python = "^3.10"
requests = "^2.28.0"

[tool.poetry.group.dev.dependencies]
black = "^23.0"
pytest = "^7.0"

[build-system]
requires = ["poetry-core"]
build-backend = "poetry.core.masonry.api"
```



#### ✅ 优点总结

- 📦 一站式：依赖、环境、构建、发布全包
- 🎯 更适合**构建 Python 库**或模块
- 🧩 使用 `pyproject.toml` 标准，兼容性强
- 📁 项目结构清晰，依赖更容易管理
- 🔁 安装和解析依赖速度快、稳定性高

#### ❌ 缺点与局限

| 缺点                   | 说明                               |
| ---------------------- | ---------------------------------- |
| 学习曲线稍陡           | 对新手不如 `venv + pip` 简单       |
| 依赖系统环境配置       | 如 Python 安装路径、PATH 变量      |
| 虚拟环境路径不可自定义 | 不如 `pipenv` 灵活（除非手动设置） |
| 不是 conda 替代品      | 不适合数据科学中非 Python 依赖管理 |

#### 🧭 适合使用 Poetry 的情况

- 👨‍💻 构建可发布的 Python 包、库、工具
- 🧪 中大型项目或团队协作（依赖清晰）
- 📦 想统一管理虚拟环境、依赖、打包流程
- ✅ 需要严格锁定依赖版本（CI/CD）

## conda

> https://anaconda.org/anaconda/conda

![image-20250507210257465](assets/image-20250507210257465.png)

### 🧪 什么是 Conda？

**Conda** 是一个**跨语言、跨平台的包管理器和环境管理器**，最初由 Anaconda 团队开发，用于 Python 生态，但现在也支持 R、Lua、C/C++、Java 等。

它主要解决两大问题：

- ✅ 包管理（安装、卸载、升级依赖包）
- ✅ 环境管理（创建和切换隔离的运行环境）

Conda 的核心目标是：

- 在**不同项目中管理不同版本的依赖和解释器**
- 管理 **Python 包 + 系统库 + C/C++ 编译依赖**（如 `numpy`、`libxml2`）
- 解决 pip 安装 C 扩展时出现的编译问题

### 🛠️ 常用命令

#### 🌱 创建和管理环境

```bash
# 创建新环境并指定 Python 版本
conda create -n myenv python=3.10

# 激活环境
conda activate myenv

# 退出环境
conda deactivate

# 删除环境
conda remove -n myenv --all

# 查看所有环境
conda env list
```

#### 📦 包管理

```bash
# 安装包
conda install numpy

# 指定源安装
conda install -c conda-forge pandas

# 卸载包
conda remove numpy

# 更新包
conda update scikit-learn
```

#### 📄 环境导出/导入

```bash
# 导出环境配置为 YAML
conda env export > environment.yml

# 从 YAML 创建新环境
conda env create -f environment.yml
```

### ✅ Conda 的优势

- 🔧 安装 C 依赖（如 `numpy`, `opencv`）无需编译
- 📦 可管理 Python 和非 Python 的依赖（如 C/C++ 库）
- 🔒 独立环境完全隔离，可重复构建
- 🌍 跨平台（Linux、Windows、macOS）
- 🔁 依赖求解算法更强，适合复杂项目

### ❌ Conda 的缺点

| 缺点                   | 说明                      |
| ---------------------- | ------------------------- |
| 体积大（Anaconda）     | 完整安装包接近 3GB+       |
| 镜像下载速度慢（国内） | 需配置清华/中科大等镜像源 |
| 环境启动慢             | 尤其在 Windows 上         |
| 部分 Python 包版本落后 | 相比 PyPI 有时版本较旧    |
| 与 pip 混用需谨慎      | 可能破坏 Conda 环境一致性 |

### 📚 推荐镜像源（国内）

```bash
# 清华镜像
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/
conda config --set show_channel_urls yes
```
