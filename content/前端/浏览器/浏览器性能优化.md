# 性能优化

性能无非是，资源加载问题、文件大小问题、代码逻辑问题、交互体验问题。 然后按这4个方向不断的定优化指标的事



# 首屏加载优化

## 性能指标

- ==FP== (First Paint, 首次绘制)

  网页首次呈现任何视觉元素的时间。

  > 在 SPA 应用中主要是第一次 dom 渲染时间，通常没有意义。

- ==FCP== (First Contentful Paint, 首次内容绘制)

  从网页开始加载到网页任何部分的内容呈现在屏幕上所用的时间。

  > 在 SPA 应用中，FP 到 FCP 中间主要是 js 执行，太慢就会导致白屏时间太长。

- ==FMP== (First Meaningful Paint, 首次有效绘制)

  主要内容呈现的时间

- ==LCP== (Largest Contentful Paint, 最大内容绘制)

  从网页开始加载到屏幕上呈现最大的文本块或图片元素所用的时间。

额外指标:

- INP (Interaction to NextPaint)：用户交互到页面响应的时间。
- TBT (Total Blocking Time)：从FCP到TTI （可交互时间）之间的阻塞时间。
- CLS (Cumulative Layout Shift)：页面意外布局偏移的累计分数。
- TTFB (Time to First Byte)：首字节到达时间。

## 优化方案

1. 图片优化

   使用 webp 格式，压缩图片和控制图片尺寸，懒加载 `loading="lazy"`

   > 在 http2 下没有必要使用雪碧图，即使在 http2 下请求多张图片也只需建立一次 HTTP 请求。[Reference](https://github.com/haizlin/fe-interview/issues/55)

2. 字体优化

   字体瘦身，例如使用 [fontmin](https://ecomfe.github.io/fontmin/) 实现字体子集化

   > 不推荐：Base64 嵌入字体会造成阻塞渲染，[font-spider](https://github.com/aui/font-spider) 已经不维护了

3. 文件压缩

   构建工具：使用 [cssnano](https://www.cssnano.cn/) 压缩 CSS，使用 [terser](https://terser.org/) 压缩 JS

   服务端：使用 Brotli 或者 Gzip

4. 代码优化

   实现组件按需加载，如 React 中的 `React.lazy` 和 `<Suspense fallback>`

   按需导入第三方库

   使用 tree-shaking，但这在打包工具中几乎都是自带。但有一个前提是 tree-shaking 仅支持 esm 模块，对于 amd, cmd, umd, commonjs, esm 等模块不支持

5. 精简三方库

   移除不必要的第三方库，对于多语言支持的库可以删除使用不到的语言。

6. 预加载

   使用 `<link rel="preload" href=x.js" as="script">`，`<link rel="preload" href="x.css" as="style">` 提前加载 Javascript，CSS，字体等。

   对非关键 Javascript 进行延迟加载 `<script defer></script>`，`<script async></script>`

7. 预渲染和静态生成

   使用 [prerenderer](https://github.com/Tofandel/prerenderer) 提前生成静态 HTML 内容，提高加载速度。

8. 启用浏览器缓存

   强缓存（Expire、Cache-Control）、协商缓存（Last-Modifed、If-Modifed-Since，Etag、
   If-None-Match）、策略缓存 service-worker

9. HTTP/2 Server Push

   配置 HTTP/2 服务器，在响应 HTML 时主动推送关键资源。

10. 使用 CDN

    用 CDN 托管静态资源

11. 服务端渲染 SSR，静态生成 SSG



## 指标获取

- 开发阶段
  1. 通过控制台中的 Performance 面板和 LightHouse 面板获取。
  2. 通过 [web-vitals](https://github.com/GoogleChrome/web-vitals) 监控 FCP, LCP, CLS 等指标。
- 生产环境
  1. 通过 Google Analytics 配置自定义事件记录指标。
  2. 通过 [Performance API](https://developer.mozilla.org/zh-CN/docs/Web/API/Performance) 获取 FP、FCP。
  3. 通过 [Mutation Observer API](https://developer.mozilla.org/zh-CN/docs/Web/API/MutationObserver) 获取 FMP。
  4. 使用开源平台(Prometheus, Grafana)或者第三方服务(New Relic, Datadog)
